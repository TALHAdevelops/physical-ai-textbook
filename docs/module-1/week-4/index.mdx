## 1. Nodes in Practice

A ROS 2 system is composed of many independent nodes. Let's create a simple publisher and subscriber node in Python.

### Python Node Structure

Every ROS 2 Python node needs a basic structure:
*   Import `rclpy` and `Node`.
*   Initialize `rclpy`.
*   Create a class that inherits from `Node`.
*   Implement a constructor to set up publishers, subscribers, timers, etc.
*   Spin the node.

```python
# simple_node.py
import rclpy
from rclpy.node import Node

class SimpleNode(Node):
    def __init__(self):
        super().__init__('simple_node')
        self.get_logger().info('SimpleNode has been started!')

def main(args=None):
    rclpy.init(args=args)
    simple_node = SimpleNode()
    rclpy.spin(simple_node) # Keep node alive
    simple_node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## 2. Topics: Publish/Subscribe Pattern

Topics are the most common way nodes exchange information in ROS 2.

### Publisher Example

Let's create a node that publishes "Hello ROS 2" messages every second.

```python
# minimal_publisher.py
import rclpy
from rclpy.node import Node
from std_msgs.msg import String # Standard string message type

class MinimalPublisher(Node):
    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10) # Topic name 'topic', QoS profile depth 10
        timer_period = 1.0  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = f'Hello ROS 2: {self.i}'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.data}"')
        self.i += 1

def main(args=None):
    rclpy.init(args=args)
    minimal_publisher = MinimalPublisher()
    rclpy.spin(minimal_publisher)
    minimal_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### Subscriber Example

Now, a node that subscribes to the 'topic' and prints received messages.

```python
# minimal_subscriber.py
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalSubscriber(Node):
    def __init__(self):
        super().__init__('minimal_subscriber')
        self.subscription = self.create_subscription(
            String,
            'topic',
            self.listener_callback,
            10)
        self.subscription  # prevent unused variable warning

    def listener_callback(self, msg):
        self.get_logger().info(f'I heard: "{msg.data}"')

def main(args=None):
    rclpy.init(args=args)
    minimal_subscriber = MinimalSubscriber()
    rclpy.spin(minimal_subscriber)
    minimal_subscriber.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### Running Publisher and Subscriber

1.  Open two terminals.
2.  In the first terminal, run the publisher: `ros2 run <your_package_name> minimal_publisher`
3.  In the second terminal, run the subscriber: `ros2 run <your_package_name> minimal_subscriber`

You should see the subscriber printing the messages from the publisher.

## 3. Services: Request/Reply Pattern

Services are used for synchronous, one-time request-response interactions.

### Service Server Example

Let's create a service that adds two integers.

```python
# minimal_service.py
import rclpy
from rclpy.node import Node
from example_interfaces.srv import AddTwoInts # A predefined service message type

class MinimalService(Node):
    def __init__(self):
        super().__init__('minimal_service')
        self.srv = self.create_service(AddTwoInts, 'add_two_ints', self.add_two_ints_callback)
        self.get_logger().info('Add two ints service ready.')

    def add_two_ints_callback(self, request, response):
        response.sum = request.a + request.b
        self.get_logger().info(f'Incoming request: a: {request.a} b: {request.b}')
        self.get_logger().info(f'Sending back response: {response.sum}')
        return response

def main(args=None):
    rclpy.init(args=args)
    minimal_service = MinimalService()
    rclpy.spin(minimal_service)
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### Service Client Example

A client to call the `add_two_ints` service.

```python
# minimal_client.py
import sys
import rclpy
from rclpy.node import Node
from example_interfaces.srv import AddTwoInts

class MinimalClientAsync(Node):
    def __init__(self):
        super().__init__('minimal_client_async')
        self.cli = self.create_client(AddTwoInts, 'add_two_ints')
        while not self.cli.wait_for_service(timeout_sec=1.0):
            self.get_logger().info('service not available, waiting again...')
        self.req = AddTwoInts.Request()

    def send_request(self, a, b):
        self.req.a = a
        self.req.b = b
        self.future = self.cli.call_async(self.req)
        rclpy.spin_until_future_complete(self, self.future)
        return self.future.result()

def main(args=None):
    rclpy.init(args=args)
    minimal_client = MinimalClientAsync()
    response = minimal_client.send_request(int(sys.argv[1]), int(sys.argv[2]))
    minimal_client.get_logger().info(
        f'Result of add_two_ints: for {minimal_client.req.a} + {minimal_client.req.b} = {response.sum}')
    minimal_client.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### Running Service and Client

1.  Open two terminals.
2.  In the first terminal, run the service server: `ros2 run <your_package_name> minimal_service`
3.  In the second terminal, run the service client: `ros2 run <your_package_name> minimal_client 5 7` (replace 5 and 7 with your desired numbers)

The client should print the sum returned by the service.
